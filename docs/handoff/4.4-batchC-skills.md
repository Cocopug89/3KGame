### One deliberate deviation from skill-trigger-design.md §8's catalog: 急救

§8 originally sketched 急救 as a `demand.open(peach)` proxy (supply a peach into *anyone's* dying
window). §11's later cross-check confirmed the simpler wording instead — "outside your own turn,
any red card is a Peach" — and the locale text (written in 4.1a, also cross-checked) says exactly
that. This lane followed §11 as the authoritative correction rather than §8's earlier draft:
`jijiu` is a plain `cardsAs` query gated on `G.seats[G.activeSeat] !== owner`, answering the *existing*
`peach` demand fold (`pump.ts`'s `demandAsk`) rather than needing any proxy machinery at all. Much
simpler, and it's what the sources actually say.

## New engine-adjacent mechanism: `content/effects/strike.ts`'s third resolve() step

铁骑 and 流离 both act on `card.target` — *before* the dodge demand is built — and both need to
change something about that demand (whether it's asked at all, or who it's asked of) that the
existing two-call `strike.ts` couldn't express: the demand+resume frames were already returned,
in one shot, by the same call that announced the target, so a `card.target` listener could only
ever reach them by the retro-edit anti-pattern §2.1 forbids.

**Fix:** `strike.ts` now has three states instead of two (`ctx.demanded` → step 3, `ctx.targeted` →
step 2, neither → step 1), checked MOST-ADVANCED FIRST so every resume ctx built before 4.4 (which
only ever set `demanded`) still lands on step 3 unchanged. Step 2 is the new synchronization point:
by the time it runs, `card.target`'s whole fan-out (including any judge or request it raised) has
fully drained, and it reads two generic turn flags — `tieji.forceHit` and `liuli.redirectTo` — set
by those two skills, clears them immediately, and only then builds the demand (or skips it
entirely). **No pump.ts/frames.ts change was needed** — this is a content-file restructure, not a
new engine mechanism, and it required updating `server/test/content/strike.test.ts`'s exact-array
assertions (the old single "first call" test became two: announce-target, then build-demand; two
new tests cover the 铁骑/流离 branches). This is the one shared file this lane's own review should
weigh most carefully.

⚠️ Documented simplification, flagged in `liuli.ts`'s own header: a redirected strike does not
re-fire `card.target` for the new target, so the new target cannot chain another 流离/铁骑 of their
own. Standard rules allow chaining; this is a scope cut for a content-only lane, not a correctness
bug in what's implemented.

⚠️ Documented simplification, flagged in `guanxing.ts`'s own header: the real skill lets Zhuge
Liang split the revealed cards between the top AND the bottom of the draw pile. `{t:'moveCards'}`'s
`drawPile` zone only supports re-inserting at the top (`unshift`); there is no "append to the
bottom" primitive. v1 lets the player reorder the top N among themselves, always back on top —
correct and useful, just not the full skill. Flagged for whichever session next extends
`Zone`/`putInZone` with a bottom-insert option.

## New bgio stages/moves (`server/src/bgio/game.ts`)

Five new request kinds, inserted after the existing `discard` stage, each following the established
shape (validate against `G.pending`, apply, clear pending, pump, resync) and — unlike `chooseCard`'s
family — each answered **directly by its move**, since none of these five needs a further resume hop:

- `guanxing` / `arrangeCards(order)` — validates a permutation of the offered ids, reorders via a
  drawPile→drawPile `moveCards` (take-then-put, same trick every other zone already uses).
- `guicaiRetrial` / `submitRetrial(cardId | null)` — discards the named card, pushes `{t:'retrial'}`.
- `yijiDistribute` / `distributeCards(assignments)` — one `moveCards` per non-self assignment.
- `liuliRedirect` / `redirectStrike(cardId, newTarget)` — discards the cost card, writes
  `G.turnFlags['liuli.redirectTo']`.
- `declareSuit` / `declareSuit(suit)` — the one case that genuinely is a plain resume-ctx answer
  (`applyToResumeFrame`), since `fanjian`'s active effect is the frame directly on top when it runs.

Also: `skillTypes.ts` gained `activeCardCount?: number` (additive, optional) and `useSkill` gained
one check for it — 结姻 costs exactly 2 cards, 离间 exactly 1, enforced generically rather than as a
skill-shaped `if` in the move.
