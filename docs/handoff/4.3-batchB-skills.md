## New engine mechanisms (files this lane does not exclusively own)

Per the finish-workflow-plan, Lane E's file footprint was meant to be `content/skills/*` +
`skillRegistry.ts` + locales only. Implementing three of these twelve skills correctly needed real
engine mechanism the design doc anticipated but hadn't built yet (§9's "if a task finds itself... stop,
that's the design breaking" is about hiding a rule inside a skill — these are generic, reusable
primitives, the same category 3.4's `{t:'reveal'}` and 3.6's `{t:'demandSupply'}` were). No other lane
was found touching `pump.ts`/`frames.ts`/`bgio/game.ts` for the same reason at the time of writing, but
**re-diff these five files against this note if anything looks missing**:

1. **`{t:'loseHp'}`** (`frames.ts`, `pump.ts`) — 苦肉's "失去1点体力" is explicitly NOT 伤害 (confirmed via
   a second source: 夏侯惇's own 刚烈 skill text calls this out — "流失体力是不算的" — HP loss without
   damage does not trigger damage-based skills). A bare `{t:'damage'}` would have wrongly fired
   奸雄/反馈/刚烈/遗计. `loseHp` decrements `hp` directly and opens the `dying` window at ≤0, exactly like
   `damage` step 2, but never touches `G.damage` or emits `damage.before`/`damage.after`. `killer` is
   always `null` (no attacker ⇒ no death-consequence bounty/penalty).
2. **`putInZone`'s equip-slot fix** (`pump.ts`) — replacing an equipped card used to
   `G.discardPile.push(replaced)` directly, with **no `card.lost` emission at all**. Harmless until 枭姬
   ("lose an equipped card ⇒ draw 2") became the first listener that needed to hear it. `putInZone` now
   returns the bumped card(s); the `'moveCards'` case pushes a second `cardLostFrame(..., 'equip')` when
   that's non-empty. Backward compatible (an unused return value everywhere else).
3. **`duel.ts`'s damage frame now carries `card: ctx.cards[0]`** — needed so 裸衣's `damage.before`
   handler can tell "this damage came from a 杀 or 决斗" apart from card-less AoE damage
   (南蛮入侵/万箭齐发, which must NOT get the +1). `card` was already optional on `{t:'damage'}`; nothing
   previously read it back off this specific frame.
4. **Two new generic request kinds** (`bgio/game.ts`): `chooseOption` (pick one of a short labelled list
   — `{id, labelKey}[]`) and `choosePlayer` (pick one living player from `candidates`, or `null` to
   decline). Both write into the top `resume` frame's ctx via `applyToResumeFrame`, exactly like
   `chooseCard`/`demandCard` do. **Added to `shared/src/threeKingdoms.ts`'s `THREE_KINGDOMS_STAGE_MOVES`**
   — skipping this step is the exact "client silently never sends the move" failure the file's own
   comment warns about, and `server/test/bgio/stages.test.ts` should now cover both.

## ⚠️ Client gap — same shape as 3.4/3.6's `chooseCard` `{z:'revealed'}` gap

`chooseOption`/`choosePlayer` are server-complete and correct but have **no `PromptPanel` renderer**. A
table that reaches 刚烈 (non-Heart judgement), 洛神 (black judgement), or 突袭 (opts in) will stall on
whichever player is asked — `promptFor()` returns `null` for an unknown request kind, which is also what
a spectator gets, so nothing *looks* wrong (CONTINUE.md's own warning: "a request kind with no
`promptFor()` case is a silently stalled table, and it has now happened twice"). This makes it a third
time. Flagged for whichever session next opens `PromptPanel`/`ChoicePanel`; the two moves already exist
(`TableActions` needs `chooseOption(optionId)` / `choosePlayer(id | null)`), same shape as the
`demandCard`/`chooseCard` cases already wired.

## Documented simplifications / rules interpretations

- **集智 (jizhi)** keys off `effectKey` membership in a hardcoded "non-delayed trick" set (everything in
  `content/standard/cards.json`'s trick list except `indulgence`/`lightning`), not the physical card's
  `type`. This is deliberate: 甘宁's 奇袭 (Batch A) lets a black BASIC card be played AS 过河拆桥, and by
  the rule text ("当你使用...锦囊牌时") that still counts as using a trick. Checking `type==='trick'`
  would have missed it.
- **制衡 (zhiheng)**: cross-checked against a second source — Sun Quan's Standard 制衡 discards to the
  discard pile and redraws, it is **not** the "place at the bottom of the draw pile" mechanic some other
  skill/print uses. No new zone-position primitive was needed.
- **苦肉 (kurou)**: cross-checked — Standard edition is **unlimited per turn**, no discard cost. The
  `界限突破` reprint (once-per-phase, +discard cost) is a different, out-of-scope card per
  three-kingdoms-plan.md.
- **刚烈 (ganglie)**'s discard-2 cost pays "what they have" if the source holds fewer than 2 cards,
  rather than erroring — same shrug `duress.ts`/`harvest.ts` already take on a target that doesn't
  fully qualify for a cost/effect by the time it resolves.
