## 3. What a future session most needs to know

The audit's §5 guard rails, in one line each:

1. A `PendingRequest` payload ships in full to one player — put only what that player is entitled to
   in it.
2. **`confirmSkill` discloses the raw `TriggerEvent` to the trigger's owner.** Safe today only because
   every optional trigger in the Standard 40 is self-scoped. A skill that listens to *another*
   player's `card.lost` would leak their card ids through this payload. **This is the next leak this
   codebase will have, if it has one.**
3. `G.log` is public — a log entry may only name a face-up card.
4. `G.turnFlags` is public — never put a card id in one.
