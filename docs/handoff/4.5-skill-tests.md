## 4. What the tests pin that is worth knowing about

1. **克己 reads `turnFlags.strikeUsedInAction`, NOT `strikesPlayed`** (design §2.2 — a 杀 played
   *in response* during your action phase counts). `keji.test.ts` has an explicit test for the
   wedge in both directions: `strikesPlayed: 0` + `strikeUsedInAction: true` ⇒ does not fire;
   `strikesPlayed: 2` alone ⇒ still fires. A "simplification" back to `strikesPlayed` fails here.
2. **反间's direction** (周瑜 gives one of his OWN cards; he does not steal one) is pinned by an
   assertion on the `moveCards` direction. This was the Opus review's correction — a regression
   fails `fanjian.test.ts` step 3.
3. **The slot protocol is a leak surface, so it gets a leak test.** `fankui.test.ts` asserts not
   only that the request carries `{z:'hand', index:n}` slots but that the serialised frames contain
   **no hand-card id at all**. This is the assertion 5.4's anti-cheat audit (Lane I) will want to
   generalise.
4. **苦肉 emits `{t:'loseHp'}` and never `{t:'damage'}`** — asserted both ways round, because the
   whole reason that primitive exists is that 奸雄/反馈/刚烈/遗计 must not fire off it.
5. **裸衣's split** (optional trigger writes the flag; locked query + mandatory `damage.before`
   trigger read it) is tested as three separate contracts, including that the damage half is
   `optional: false` — an accidental `true` there is the "eleven prompts a turn" failure mode §3.4
   warns about.
6. **遗计's limit is `once_per_damage`, not `once_per_turn`** — asserted directly, since the two
   are indistinguishable in a Standard game (all hits are 1 point) and only diverge later.
7. **观星 never mutates `G`** — asserted that `G.drawPile` is unchanged after `effect()` and that
   the frames are a lone `request` with no `moveCards`. Its v1 simplification (reorder the top N;
   no bottom-insert) is pinned as implemented, per 3.7's precedent of documenting a gap rather than
   papering over it.

## 5. Things found but NOT fixed (test-only lane — flagged, not patched)

1. **集智's non-delayed-trick set omits `nullification`.** `jizhi.ts` lists nine effectKeys;
   无懈可击 is a non-delayed trick and by the plain rule text ("使用非延时类锦囊牌") should draw a
   card too. Whether it is *reachable* depends on whether the nullify path emits `card.play` at all
   (3.2's `nullifyWindow` supplies the card through the demand protocol, not `playCard`), so this
   may be moot in practice — but the set is a hard-coded list, and a reader will not know that from
   looking at it. `jizhi.test.ts` pins the nine keys as implemented and this note is the record.
   A one-line fix belongs in `jizhi.ts` (4.3's file), not in a test.
2. **离间's synthesized 决斗 does not fire 集智** (it bypasses `{t:'play'}`, so no `card.play` is
   emitted) — already documented in `lijian.ts`'s own header as an accepted simplification;
   repeated here because it is now *cross-batch* and the two files that would tell you are far
   apart. Pinned in `lijian.test.ts` as "no `{t:'play'}` frame", which is the property both the
   unnullifiability (wanted) and the missing 集智 (accepted) fall out of.
